Lever 01 / 05 — U-curve

The U-Curve.▋

Models attend most to the start and end of a prompt, least to the middle, so front-load it and back-load it.
u-curve.txt
$ prompt ›
TD Builders Club S01 / 08 — Lever 01 · U-curve From your COO interns
Human analog first: if I'm telling Wamiq something important, I start with the key stuff and end with the warnings. Same trick. "Lost in the middle" has a name and there's research. Restating the ask at the bottom feels redundant — it isn't.
Lever 02 / 05 — Escape hatch

Give it an out.▋

Models default to confident answers. Make "I don't know" a legal move and you stop getting confident lies.
vendor-pick.txt
$ prompt ›
TD Builders Club S01 / 08 — Lever 02 · Escape hatch From your COO interns
Make it a hater. The model's job is to find the weakness, not bless your picks. Same task, one extra clause — completely different model behavior.
Lever 03 / 05 — Stop condition

Tell it when to stop.▋

Without a stop condition, agents loop. Spell out the exit so the model knows what "done" looks like.
agent.txt
$ agent ›
TD Builders Club S01 / 08 — Lever 03 · Stop condition From your COO interns
Without this, your agent burns $40 of tokens and ships the wrong answer. Stop conditions are the cheap part of agent design — and the part everyone skips. Always: budget · success criteria · failure ceiling.
Lever 04 / 05 — Explore vs build

Explore first. Build second.▋

Models jump to the first plausible answer. State the mode out loud — explore or build — and you get options, not commitments. This is especially important when designing architecture!
explore.txt
$ prompt ›
TD Builders Club S01 / 08 — Lever 04 · Explore vs build From your COO interns
First idea is rarely the best. The model's first idea is even less so — and once it commits to code, it argues for the code. "Don't build yet" is the magic clause. Try it on your next ticket.
Lever 05 / 05 — Meta-prompting · handoffs

Hand off in a doc, not a chat.▋

Drop a claude.md for Claude Code, a design.md for GitHub Copilot, an M365 brief for Copilot Studio. The model loads it every session. The chat dies. The doc lives.
claude.md · repo root
$ cat claude.md ›
TD Builders Club S01 / 08 — Lever 05 · Meta-prompting From your COO interns
Once you write claude.md, every Claude session starts informed. Same idea for M365 — drop a design.md in the SharePoint and Copilot Studio picks it up. Stop re-explaining the project on every chat.

Safety net

Stuck? We got you.

01
Ask the model to explain its own error.
Paste the stack trace back in. "Walk me through this." 70% of the time, that's it.
02
Reset the chat.
Long threads poison context. Start a fresh one with just the snippet that matters.
03
Flag a mentor. ●
We'll be floating around to help with anything! This is the one we're here for.
04
Shrink the idea.
Cut the feature in half. Then in half again. A two-line script is a finished project.
05
Pair up.
Find someone who's stuck on something different. Trade brains for 10 minutes.
▸ Tonight's win: one prompt you improved, and you can say why. Shipping is bonus.
